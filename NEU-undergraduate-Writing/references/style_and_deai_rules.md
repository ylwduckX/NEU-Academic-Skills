# 中文论文文风与降 AI 味规则

本文件只约束正文表达，不约束排版。编辑 Word/Markdown 草稿时，保护引用编号、公式编号、图表编号、术语、变量名和已有占位符。

## 1. 零捏造原则

降 AI 味只能改表达，不能新增事实。

禁止新增：

- 数据、指标、百分比、排名、显著性结论。
- 实验设置、数据集、硬件、超参数。
- 引用、作者、年份、DOI。
- 贡献点、应用价值、系统功能。

无法核查的内容标记为 `【待补证】`。

## 2. 信息密度优先

每句话都应传递可核查信息。删除不能回答“具体是什么”的话。

| 弱表达 | 改写方向 |
|---|---|
| 具有重要意义 | 说明对哪个任务、系统或应用有何作用 |
| 取得显著提升 | 写具体指标和数值；没有数值则降级 |
| 进行了全面研究 | 列出研究的对象、范围和方法 |
| 有效解决问题 | 写解决了哪个子问题、证据是什么 |

## 3. 克制措辞

避免无证据的绝对化表达：

- 显而易见、毋庸置疑、必然、完全、从根本上、全面解决。
- 领先、首次、最优、显著、大幅，除非证据足够。
- “证明了”慎用；实验通常写“表明”“支持”“验证了”。

可用限定语：

- 在本文实验设置下。
- 从当前结果看。
- 在所选数据集上。
- 相比所选基线。
- 初步说明。

## 4. 常见 AI 痕迹

高优先级修复：

- 模板开头：“近年来”“随着……快速发展”“在当今社会”。
- 空话形容词：“重要”“有效”“全面”“显著”“优秀”。
- 填充连接词：“总之”“综上所述”“值得注意的是”“需要指出的是”“不难发现”。
- 机械排比：连续多句以“本文提出/本文设计/本文实现”开头。
- 模糊量化：“大量研究”“众多方法”“多次实验”“大部分情况”。
- 堆叠引用：一句话中多个引用并列，但没有分析每篇文献的贡献或局限。

## 5. 分章节降 AI 要点

| 章节 | 优先修复 |
|---|---|
| 摘要 | 删除泛泛背景；补方法关键词、验证方式和关键结果 |
| 绪论 | 用具体应用和研究空白替代宏大背景 |
| 相关工作 | 按主题比较，避免作者年份流水账 |
| 方法 | 写清输入输出、设计动机、实现细节，不夸效果 |
| 实验 | 写具体数据集、基线、指标、结果，不只说“有效” |
| 结论 | 回答研究问题，承认局限，不引入新结果 |

## 6. 改写流程

1. 先找空话、绝对化、模板句和堆叠引用。
2. 删除不能被证据支持的形容词。
3. 用具体对象、条件、指标、图表或引用替换模糊表达。
4. 拆分超过一屏难读的长句，补足主语。
5. 检查改写后是否新增事实；若新增，撤回或标 `【待补证】`。

## 7. 保真要求

- 不改引用编号，如 `[1]`、`[3-5]`、`[CITE:关键词]`。
- 不改公式编号、图号、表号。
- 不随意替换已经定义的术语和缩写。
- 不把项目级 Markdown 注释、TODO、占位图表误删。
- 不引入任何非 NEU 的排版要求。
