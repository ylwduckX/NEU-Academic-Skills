# 论文内容质量通用标准

本文件约束论文内容质量与行文判断，不约束学校排版、页边距、字体、表格样式或 Word 模板。具体章节骨架优先服从项目目录中的论文大纲、导师要求和 NEU 排版 skill。

## 1. 一句话主线

写作前先确认一句话主线：

> 本文针对【问题/场景】中的【关键困难】，提出/实现/验证了【核心方法或系统】，并通过【证据类型】证明【主要效果或价值】。

如果无法写出这句话，不要直接生成长段正文。先在草稿说明中标出缺失项，例如 `【待确认：核心贡献】`、`【待补证：实验结果】`。

## 2. 三个支柱

学位论文正文必须反复服务三个问题：

| 支柱 | 写作要求 | 常见失败 |
|---|---|---|
| 做了什么 | 明确本文完成的工作、方法、系统、实验或分析 | 只写背景，不写本文工作 |
| 为什么可信 | 给出引用、代码事实、公式推导、实验结果、消融、图表或实现细节 | 只有“有效”“显著”等判断 |
| 为什么重要 | 说明该工作解决了哪个具体困难，对任务、系统或应用有什么价值 | 空泛拔高“具有重要意义” |

## 3. 摘要五个动作

摘要不是背景介绍，应完成五个动作：

1. 交代核心成果。
2. 说明问题为什么困难且值得研究。
3. 概括方法或系统的关键思路。
4. 给出验证方式。
5. 写出最关键的结果或结论。

缺少量化结果时，不得编造数字；写作时标注 `【待补实验结果】`。

## 4. 标题后导语

除摘要、目录、参考文献、致谢等特殊部分外，章、节、小节标题后应先有导语，再进入列表、图表、公式、算法或下一级标题。

导语至少完成四件事：

- 指明本层级讨论什么。
- 说明为什么在这里讨论。
- 承接上文或论文主线。
- 预告内部展开顺序。

## 5. 读者期望原则

- 一段一义：每段只服务一个主张。
- 旧信息在前，新信息在后：先承接已有内容，再引入新概念。
- 重点置尾：把本句最重要的信息放在句末。
- 主谓邻近：避免主语和谓语隔得太远。
- 先定义后使用：术语、缩写、符号首次出现时必须解释。
- 先解释后展示：公式、图表、算法前先说明用途。
- 显式过渡：段落之间说明递进、转折、因果或总结关系。

## 6. 跨章节闭合

绪论提出的研究问题、贡献和挑战，必须在实验、结论或讨论中得到回应。

- 绪论声明了贡献，实验中应有对应证据。
- 方法章节提出了模块，实验中应有主结果、消融、可视化或复杂度分析支撑。
- 相关工作推出了研究空白，本文方法应明确回应该空白。
- 结论应回答绪论中的问题，不引入新实验、新引用或新贡献。

## 7. 证据不足时的处理

证据不足时要降级表达：

| 情况 | 正确处理 |
|---|---|
| 没有实验数值 | 使用 `【待补实验结果】`，不写“提升 X%” |
| 没有引用 | 使用 `[CITE:关键词]` 或 `【待补引用】` |
| 只有单组实验 | 写“在本文设置下”，不写“充分证明” |
| 结果不显著 | 写“表现接近/具有竞争力/在部分指标上改善” |
| 贡献边界不清 | 写“本文主要完成了……”，不写“首次/领先/全面解决” |
