逐项打 ☑/☒/⚠ 并给定位（页码/章节/行号）。

### 1.1 整体结构
- [ ] 封面使用学校模板
- [ ] 含郑重声明页
- [ ] 中英文摘要齐全
- [ ] 目录自动生成、层级正确
- [ ] 正文章节齐全（绪论/相关工作/方法/实验/结论）
- [ ] 参考文献章节齐全

### 1.2 字体字号行距
- [ ] 正文宋体小四（英文 Times New Roman）
- [ ] 章标题黑体小二
- [ ] 二级标题黑体四号
- [ ] 三级标题黑体小四

- [ ] 章标题段前 0.8 行段后 0.5 行

### 1.3 页面与页码
- [ ] A4 + 上下 2.5 左 3.0 右 2.5 cm
- [ ] 页眉「东北大学本科生毕业设计（论文）」+ 章节名 + 分隔线

### 1.4 图表公式
- [ ] 图按章编号「图 X.Y」，标题居中于图下方
- [ ] 表按章编号「表 X.Y」，标题居中于表上方，三线表
- [ ] 公式按章编号「(X.Y)」，居右
- [ ] 图内文字简体中文 + 五号
- [ ] 图表无跨页

### 1.5 参考文献
- [ ] GB/T 7714-2005 顺序编码格式
- [ ] 正文「[N]」上标
- [ ] 多引用 [2,4,7] 或 [4-7]
- [ ] 标题中无引文标注
- [ ] 每条核验通过（参考 Writing skill 的核验日志）

### 1.6 形式审查表对照
按 `templates/形式审查表.docx` 中每一项打勾；缺失项明确指出。

## 维度 2：文字与行文细则

- 错别字：用上下文 + 常见混淆字典（"的/地/得"、"的"作助词时不用"地"等）扫描。
- 病句：长句拆解后判定主谓宾完整性；列出「悬空主语 / 双重否定 / 时态错位」。
- 学术腔失当：禁词清单 ——「众所周知 / 显而易见 / 毋庸置疑 / 不言而喻」；模糊量词 ——「显著 / 极大 / 大幅 / 非常」无数据支撑直接标记。
- 逻辑断裂：定位「然而 / 因此 / 综上」前后逻辑跳跃处。
- 章节衔接：每章首段是否承接上章末段。
- 标题后导语：章、节、小节标题后是否先说明本层级讨论对象、写作目的、与上文关系和展开顺序。
- 段落功能：核心段落是否至少包含主张、证据、解释或意义；纯背景句堆叠需标记。
- 调用 `paper-audit` 拿到深度评审结论后整合。

## 维度 3：创新点 + 综合质量

### 3.1 创新点对照（基于 context.md）
对每个声称的创新点：
- 真创新：方法/视角/数据集/任务在文献中无同等设计 → ✅
- 增量改进：基于已有方法的工程优化或参数微调 → ⚠
- 复述他人：本文方法等价或近似已发表工作 → ❌（红色高危）

### 3.2 质量评价表预评分（附件 17 工学）
按表中每项指标（选题 / 文献综述 / 方法学 / 工作量 / 实验 / 结论 / 撰写规范 / 创新性 / 应用价值 …）给「优 / 良 / 中 / 合格 / 不合格」预评，并指出扣分位置。

### 3.3 评阅表预评分（附件 13 评阅教师视角）
模拟评阅教师视角，按表中分项给评语建议；指出"评阅老师容易追问"的薄弱处。

### 3.4 内容质量与论证链

按需对照 `../NEU-undergraduate-Writing/references/content_quality_standards.md` 和 `../NEU-undergraduate-Writing/references/argument_evidence_rules.md`。

#### 3.4.1 一句话论文主线

- [ ] 正文能否被概括为“问题/场景 -> 关键困难 -> 本文方法/系统 -> 证据 -> 主要效果”。
- [ ] 摘要、绪论、方法、实验、结论是否服务同一条主线。
- [ ] 是否出现多个互不支撑的贡献方向。

#### 3.4.2 贡献-证据映射

对每个贡献或关键主张建立审查表：

| 主张/贡献 | 正文位置 | 证据位置 | 证据类型 | 风险 |
|---|---|---|---|---|

风险判定：

- ✅ 低风险：有实验、引用、代码事实、公式或图表支撑。
- ⚠ 中风险：有部分证据，但缺少关键设置、数值、对照或引用。
- ❌ 高风险：只有结论，没有证据。

#### 3.4.3 相关工作质量

- [ ] 是否按主题、技术路线或问题类型组织，而非作者年份流水账。
- [ ] 每个主题组是否有比较、让步、局限或总结。
- [ ] 相关工作末尾是否自然推出研究空白。
- [ ] 研究空白是否被本文方法回应。

#### 3.4.4 方法章节深度

- [ ] 每个模块是否说明动机、设计、实现和预期作用。
- [ ] 是否解释为什么采用该设计，而不只是“本文采用 X”。
- [ ] 关键术语、输入输出、变量和符号是否先定义后使用。
- [ ] 方法效果是否被提前夸大，抢占实验章节职责。

#### 3.4.5 实验-主张对应

- [ ] 每个实验是否说明验证哪个主张。
- [ ] 主结果是否支撑整体有效性。
- [ ] 消融、可视化、复杂度、失败案例等是否服务具体判断，而非装饰性堆砌。
- [ ] 结果解读是否包含趋势、差异和可能原因。
- [ ] 证据不足时是否降级表达，而非写“充分证明”“显著优于”。

#### 3.4.6 绪论-结论闭合

- [ ] 绪论提出的研究问题是否在结论中被回答。
- [ ] 绪论贡献是否在实验或结论中逐项回应。
- [ ] 结论是否新增了正文没有支撑的新贡献、新实验或新结论。

## 维度 4：查重 / AIGC

### 4.1 API 路径（review_config.json 含可用 key 时）

样板 `config.example.json`：
```json
{
  "cnki":     {"endpoint": "https://...", "key": ""},
  "turnitin": {"endpoint": "https://...", "key": ""},
  "wanfang":  {"endpoint": "https://...", "key": ""},
  "gptzero":  {"endpoint": "https://...", "key": ""},
  "zhuque":   {"endpoint": "https://...", "key": ""}
}
```

任一字段同时有 `endpoint` 与 `key` → 视为可用，调对应 API。  
python 命令从 `env.json` 取，默认 `conda run -n dfv python`。  
报告中附原始返回（截短到 5KB）+ 命中段落定位。

### 4.2 Claude 内评估路径（review_config.json 全空 / 缺失时）

#### 4.2.1 内容级查重启发式
- 与训练语料中已知公开文献（CVPR / ICCV / ECCV / NeurIPS / TPAMI 等本领域常见会议期刊）做术语 + 表述模式比对；
- 命中标记「⚠ 与 [作者 年份] 表述高度相似」并给出建议改写方向；
- 不输出"具体相似率百分比"（无法严谨给出）。

#### 4.2.2 AIGC 启发式
- 句长方差过低（< 5）→ AI 嫌疑；
- 连接词密度高（"首先/其次/此外/综上"反复堆叠）→ AI 嫌疑；
- 同义堆叠（"重要且关键的核心问题"）→ AI 嫌疑；
- 段落开头模板化（"近年来 / 随着 X 的快速发展"反复出现）→ AI 嫌疑；
- 空话形容词密集（"显著 / 全面 / 有效 / 重要"无数据支撑）→ AI 嫌疑；
- 堆叠式并列引用（一句话中 3 个及以上引用但无逐篇分析）→ AI 嫌疑；
- 连续 3 句以上相同句式开头（如"本文提出/本文设计/本文实现"）→ AI 嫌疑；
- 给段编号 + 风险等级（高/中/低）。

#### 4.2.3 改写建议
- 提供 1-2 条具体替换示例；
- 不推荐外部工具或平台；
- 提示用户可用 `48-copaper-ai-chinese-de-aigc`（已在 Writing skill 中链路化）做章节级改写。

## 总体结论模板

```
- 高优修订（必改）：N 项 ……
- 中优修订（建议改）：N 项 ……
- 低优修订（可选）：N 项 ……
- 总体可提交度：A / B / C / D
```

高优修订必须优先列出会影响毕业论文评价的内容问题，例如贡献无证据、研究空白未推出、实验不支撑主张、结论拔高、引用未核验、疑似大段 AI 模板句。
