# 四维审查评分细则（Review skill 按需 Read）

依据：`templates/规范.docx` + `templates/形式审查表.docx`。  
逐项打 ☑/☒/⚠ 并给定位（页码/章节/行号）。

### 1.1 整体结构
- [ ] 封面使用学校模板
- [ ] 含郑重声明页
- [ ] 中英文摘要齐全
- [ ] 目录自动生成、层级正确
- [ ] 正文章节齐全（绪论/相关工作/方法/实验/结论）
- [ ] 参考文献章节齐全

### 1.2 字体字号行距
- [ ] 正文宋体小四（英文 Times New Roman）
- [ ] 章标题黑体小二
- [ ] 二级标题黑体四号
- [ ] 三级标题黑体小四
- [ ] 行距 1.5 倍
- [ ] 章标题段前 0.8 行段后 0.5 行

### 1.3 页面与页码
- [ ] A4 + 上下 2.5 左 3.0 右 2.5 cm
- [ ] 页眉「东北大学本科生毕业设计（论文）」+ 章节名 + 分隔线
- [ ] 摘要起大写罗马页码、正文起阿拉伯

### 1.4 图表公式
- [ ] 图按章编号「图 X.Y」，标题居中于图下方
- [ ] 表按章编号「表 X.Y」，标题居中于表上方，三线表
- [ ] 公式按章编号「(X.Y)」，居右
- [ ] 图内文字简体中文 + 五号
- [ ] 图表无跨页

### 1.5 参考文献
- [ ] GB/T 7714-2005 顺序编码格式
- [ ] 正文「[N]」上标
- [ ] 多引用 [2,4,7] 或 [4-7]
- [ ] 标题中无引文标注
- [ ] 每条核验通过（参考 Writing skill 的核验日志）

### 1.6 形式审查表对照
按 `templates/形式审查表.docx` 中每一项打勾；缺失项明确指出。

## 维度 2：文字与行文细则

- 错别字：用上下文 + 常见混淆字典（"的/地/得"、"的"作助词时不用"地"等）扫描。
- 病句：长句拆解后判定主谓宾完整性；列出「悬空主语 / 双重否定 / 时态错位」。
- 学术腔失当：禁词清单 ——「众所周知 / 显而易见 / 毋庸置疑 / 不言而喻」；模糊量词 ——「显著 / 极大 / 大幅 / 非常」无数据支撑直接标记。
- 逻辑断裂：定位「然而 / 因此 / 综上」前后逻辑跳跃处。
- 章节衔接：每章首段是否承接上章末段。
- 调用 `paper-audit` 拿到深度评审结论后整合。

## 维度 3：创新点 + 综合质量

### 3.1 创新点对照（基于 context.md）
对每个声称的创新点：
- 真创新：方法/视角/数据集/任务在文献中无同等设计 → ✅
- 增量改进：基于已有方法的工程优化或参数微调 → ⚠
- 复述他人：本文方法等价或近似已发表工作 → ❌（红色高危）

### 3.2 质量评价表预评分（附件 17 工学）
按表中每项指标（选题 / 文献综述 / 方法学 / 工作量 / 实验 / 结论 / 撰写规范 / 创新性 / 应用价值 …）给「优 / 良 / 中 / 合格 / 不合格」预评，并指出扣分位置。

### 3.3 评阅表预评分（附件 13 评阅教师视角）
模拟评阅教师视角，按表中分项给评语建议；指出"评阅老师容易追问"的薄弱处。

## 维度 4：查重 / AIGC

### 4.1 API 路径（review_config.json 含可用 key 时）

样板 `config.example.json`：
```json
{
  "cnki":     {"endpoint": "https://...", "key": ""},
  "turnitin": {"endpoint": "https://...", "key": ""},
  "wanfang":  {"endpoint": "https://...", "key": ""},
  "gptzero":  {"endpoint": "https://...", "key": ""},
  "zhuque":   {"endpoint": "https://...", "key": ""}
}
```

任一字段同时有 `endpoint` 与 `key` → 视为可用，调对应 API。  
python 命令从 `env.json` 取，默认 `conda run -n dfv python`。  
报告中附原始返回（截短到 5KB）+ 命中段落定位。

### 4.2 Claude 内评估路径（review_config.json 全空 / 缺失时）

#### 4.2.1 内容级查重启发式
- 与训练语料中已知公开文献（CVPR / ICCV / ECCV / NeurIPS / TPAMI 等本领域常见会议期刊）做术语 + 表述模式比对；
- 命中标记「⚠ 与 [作者 年份] 表述高度相似」并给出建议改写方向；
- 不输出"具体相似率百分比"（无法严谨给出）。

#### 4.2.2 AIGC 启发式
- 句长方差过低（< 5）→ AI 嫌疑；
- 连接词密度高（"首先/其次/此外/综上"反复堆叠）→ AI 嫌疑；
- 同义堆叠（"重要且关键的核心问题"）→ AI 嫌疑；
- 段落开头模板化（"近年来 / 随着 X 的快速发展"反复出现）→ AI 嫌疑；
- 给段编号 + 风险等级（高/中/低）。

#### 4.2.3 改写建议
- 提供 1-2 条具体替换示例；
- 不推荐外部工具或平台；
- 提示用户可用 `48-copaper-ai-chinese-de-aigc`（已在 Writing skill 中链路化）做章节级改写。
